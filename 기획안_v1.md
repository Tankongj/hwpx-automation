## 1. 프로젝트 개요

### 1.1 목적

한글(HWPX) 문서를 AI 보조로 자동 작성하는 **Windows 데스크톱 애플리케이션**.

### 1.2 핵심 사용 시나리오

```
① 사용자: .txt 원고 업로드 (약간의 계층 기호 포함)
② 사용자: 템플릿 선택 (기본 또는 업로드한 공고 양식)
③ 프로그램: 원고 계층 분석 (정규식 → Gemini 보조)
④ 프로그램: 템플릿 스타일 매핑 (기본 10단계 또는 업로드 템플릿)
⑤ 프로그램: HWPX 생성 + 네임스페이스 수정 + 검증
⑥ 사용자: 미리보기로 확인 후 저장
```

### 1.3 타깃 사용자

공무원, 행정사, 법무사, 변호사 등 **한글 문서를 반복 작성하는 사무직 전문가군**.

### 1.4 비즈니스 모델

- **MVP**: 로컬 도구. 무료. 광고·회원제 없음.
- **상업화 단계** (MVP 이후):
  - 회원제 무료 + 광고 모델
  - 매출: Google Display Ads, 쿠팡 파트너스 등 광고 매출
  - MVP 코드에 **광고 자리 placeholder**만 미리 확보

## 4. 핵심 모듈 상세 설계

### 4.1 원고 파서 (regex_parser.py)

**입력**: 원고 .txt 텍스트(utf-8)  
**출력**: `List[Block]` (IR)

**IR Block 스키마** (`parser/ir_schema.py`)
```python
from dataclasses import dataclass
from typing import Literal, Optional

@dataclass
class Block:
    level: int            # 1~10, 본문은 0, 제목은 -1 (문서 제목)
    symbol: str           # 원문 선두 기호 (예: "Ⅰ.", "(1)", "□", "-")
    text: str             # 기호 제외한 본문 텍스트
    raw_line: str         # 원문 라인 그대로
    line_no: int          # 원고 내 줄 번호 (디버깅용)
    ambiguous: bool = False   # Gemini 해석이 필요한 경우 True
    parent_level: Optional[int] = None   # 계층 관계 (해석 후 채움)
```

**기호 → 레벨 매핑 규칙** (결정론적)
```python
RULES = [
    # (정규식, 레벨, 비고)
    (r"^#\s+(.+)$",                1,  "마크다운 H1 = Ⅰ. 대장"),
    (r"^##\s+(.+)$",               2,  "마크다운 H2 = 1 절"),
    (r"^###\s+(.+)$",              3,  "마크다운 H3 = 1) 소절"),
    (r"^\s*\(\d+\)\s*(.+)$",       4,  "(1) 단락 제목"),
    (r"^\s*[①-⑳]\s*(.+)$",        5,  "원숫자"),
    (r"^\s*□\s*(.+)$",             6,  "대주제"),
    (r"^\s*[❍◯○]\s*(.+)$",        7,  "중주제"),
    (r"^\s*-\s+(.+)$",             8,  "하이픈 글머리"),
    (r"^\s*·\s*(.+)$",             9,  "가운뎃점"),
    (r"^\s*[*※]\s*(.+)$",          10, "주석/참고"),
    (r"^---+$",                    None, "구분선, 무시"),
    (r"^\s*$",                     None, "빈 줄, 무시"),
]
```

**문서 제목 처리**  
원고 최상단의 유일한 `# 제안서` 패턴(Ⅰ./Ⅱ. 등 로마숫자 없음) → `level=-1`로 구분해 표지용 처리.

**본문(body) 판단**  
어떤 규칙에도 매치 안 되는 비어있지 않은 줄 = `level=0` 본문. 직전 제목의 하위에 붙음.

**애매 케이스 (ambiguous=True로 마킹)**
- 기호 매치됐으나 본문 길이가 50자 이상인 긴 줄 (기호를 잘못 찍은 본문일 가능성)
- 연속된 레벨 점프 (1 → 5 같은 급격한 점프)
- 기호 없는 줄이지만 짧고 명사구로 보이는 줄 (레벨 1~4 제목 후보)

### 4.4 템플릿 분석기 (template_analyzer.py)

HWPX = ZIP + XML 구조. `Contents/section0.xml`, `Contents/header.xml` 을 lxml로 파싱.

**추출 대상**
- `<hh:charPr>` 목록 → 폰트, 크기
- `<hh:paraPr>` 목록 → 들여쓰기, 문단간격
- `<hh:style>` 목록 → 스타일 정의
- `<hh:pagePr>` → 페이지 설정

**스타일맵 매칭 전략**
- 분석된 스타일들을 레벨 1~10에 매핑 시도
- 1차: 폰트 크기 기반 매칭 (20pt → 레벨1, 18pt → 레벨2/3 등)
- 2차: 스타일 이름 휴리스틱 (있다면)
- 3차: Gemini 보조 (선택) — 템플릿 스타일 목록과 우리 레벨 스펙을 주고 매핑 요청

### 4.6 검증기 (verify_hwpx.py)

- [ ] 레벨 1 대장 개수 (원고의 `# Ⅰ.` 개수와 일치?)

### 8.1 정량제안서

정량제안서는 `[서식 1]`, `[서식 2]` 같은 **양식 채우기** 구조. 파서가 완전히 다름.

### 8.2 제출서류 체크리스트

**파일명 날짜 패턴 (1차)**
```python
PATTERNS = [
    r"_(\d{8})\.",           # _20260315.
    r"_(\d{4}-\d{2}-\d{2})", # _2026-03-15
    r"_(\d{4}\.\d{2}\.\d{2})", # _2026.03.15
    r"(\d{6})_",             # 260315_
]
```

**OCR Fallback (2차)**
- Tesseract (kor.traineddata 한국어 팩)
- PDF는 pdfplumber로 텍스트 먼저 시도 → 실패 시 OCR

**필수서류 목록 생성**
- RFP/공고문 → Gemini로 "이 공고의 필수 제출서류 목록을 추출하라" 프롬프트
- 결과를 체크리스트로 변환

4. **파서 작성**: `regex_parser.py` + 샘플 원고로 유닛 테스트
5. **기본 변환 콘솔 검증**: GUI 없이 `python -m src.cli input.txt output.hwpx`로 먼저 E2E 성공 확인
6. **이후 GUI 추가**: 콘솔이 정상이면 PySide6로 감싸기

**주의사항**
- v1 코드를 복사할 때 라이선스 표기(MIT) 유지
- Gemini API 키는 절대 코드에 하드코딩 금지 → keyring/암호화 저장
- 광고 관련 코드는 MVP에서 **placeholder만**, 실제 ad network SDK 통합 금지 (별도 단계)
- 커밋 단위는 기능 단위로 작게 자주
- 첫 릴리즈 전까지 버전: 0.1.0-dev → MVP DoD 통과 시 0.1.0

## 10. 결정 근거 요약 (Why 모음)

- **새 코드베이스**: v1의 35개 스크립트는 실험용 CLI. 상업 제품으로 가려면 패키지 구조·테스트·빌드·UI가 필요하며 이는 새로 설계하는 편이 빠름. 단, 검증된 XML 조작 로직(`fix_namespaces`, `md_to_hwpx`)은 그대로 재활용.
- **정규식 + Gemini 하이브리드**: 사용자 제공 스타일 스펙이 한/글 Ctrl+1~0 체계와 정확히 일치해 기호→레벨 매핑이 결정론적. Gemini는 진짜 애매한 5~10%에만 투입 → 문서당 호출 1회, 비용 ~3원.
- **PySide6**: PyQt는 상업 유료. Flet은 신생이라 PyInstaller 성숙도·AI 코딩 지원·광고 통합 모두 아직 부족. PySide6는 LGPL + QWebEngineView + 성숙한 생태계 조합이 상업화에 가장 안전.
- **pyhwpx 제외**: 한컴 Automation SDK는 **상업 이용 시 유료 라이선스**. 광고 수익 = 상업 이용. 따라서 기본 파이프라인에서 제외. 사용자가 본인 PC에 한/글 있을 때만 옵션으로 켜는 방식이 유일하게 합법적.
- **정량·제출서류는 MVP 제외**: 정성제안서 모드만으로도 Q12 DoD가 4만자 원고 처리. 정량은 완전히 다른 파서/UI라 독립 스프린트. 너무 넓히면 MVP 출시 늦어짐.
